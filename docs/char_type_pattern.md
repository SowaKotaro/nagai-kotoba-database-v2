# char_type_pattern 変換仕様

`words.surface`（表層形）を1文字ずつ「文字タイプ」に写像した文字列。
検索・分類の軸に使う。生成は Ruby 側（[`app/models/char_type_pattern.rb`](../app/models/char_type_pattern.rb)）で行い、
`Word` の `before_validation` で自動セットする（手入力させない）。

## 写像ルール
1文字ごとに、下記を**上から順に判定し最初に一致したもの**を採用する。

| 文字タイプ | 記号 | 判定 | 例 |
| --- | --- | --- | --- |
| 漢字 | `漢` | `\p{Han}` | 殺, 人, 事, 件 |
| ひらがな | `あ` | `\p{Hiragana}` | あ, き, ゃ |
| カタカナ | `ア` | `[\p{Katakana}ーｰ]`（半角カナ・長音符を含む） | カ, ﾎ, ー, ｰ |
| 英字 | `A` | `[A-Za-zＡ-Ｚａ-ｚ]`（全角・半角ラテン文字） | A, z, Ａ |
| それ以外 | `@` | 上記いずれにも該当しない | 数字, 記号, 空白, 絵文字 |

## 具体例
| surface | char_type_pattern |
| --- | --- |
| `ABC殺人事件` | `AAA漢漢漢漢` |
| `カレーライス` | `アアアアアア`（`ー` もカタカナ扱い） |
| `令和6年` | `漢漢@漢`（`年` は漢字、`6` は数字なので `@`） |
| `Ｗｅｂ３.０` | `AAA@@@`（全角数字・記号・`.` は `@`） |

## 判定の注意点（境界・仕様上の割り切り）
- **数字は漢/あ/ア/英字のいずれでもない**ため、半角・全角ともに `@`。
- **長音符 `ー`(U+30FC) / 半角長音符 `ｰ`(U+FF70)** は Unicode 上カタカナに分類されないが、
  カタカナ語の伸ばす音として頻出するため**カタカナ(`ア`)として扱う**（`カレー` → `アアア`）。
- **半角カタカナ**（`ｱ`〜）は `\p{Katakana}` に一致するため `ア`。
- **繰り返し記号 `々`(U+3005)** は `\p{Han}` に一致するため `漢`（漢字扱い）。
  一方 `〆`(U+3006) などは一致せず `@`。
- 記号・空白・絵文字はすべて `@`。
- `nil` は空文字として扱い、結果は空文字列。

判定順を変えると結果が変わりうるため、`CharTypePattern::RULES` の順序を維持すること。
