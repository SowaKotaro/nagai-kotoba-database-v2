# デザインシステム仕様「知識アーカイブ（Knowledge Archive）」（長い言葉のデータベース）

> UI を作る/変える、または雰囲気・トーンを確認するときは**必ず本書を読む**。`CLAUDE.md` からも参照。
> トークン名・数値は確定値として扱い、勝手にデフォルトの美意識へ回帰しない。

> **改訂履歴**: 2026-07 に旧「活字見本帖（Type Specimen）」を部分改訂。オーナー判断で**採用したもの／元に戻したもの**は次のとおり。
>
> | | 内容 |
> |---|---|
> | 採用 | 色・書体トークンの刷新／ヘッダー 64px・常設検索の撤去／**紙色の奥付フッター**（墨ベタ廃止）／**検索を主役にしたホーム**／**一覧のリスト UI**／ボタン・入力の白地＋4px 角丸／セクション間 80px |
> | 差し戻し | **紙幅は 1040px**（1200px は間延びしたため）／**単語詳細は活字見本帖の文法のまま**（墨 2px の下端罫・墨 1px 枠の語義パネル・黒ベタの番号刻印・墨枠のタグ/チップ/シェアボタン） |
>
> つまり「**ホームと一覧は静かな知識アーカイブ、単語詳細は活字見本帖の見せ場**」という二段構えが現行の確定仕様。

---

## 0. 思想（1行）

**「日本語という文化財を展示するギャラリー」**。UI をデザインするのではなく、言葉を静かに展示する額縁を設計する。

- 目指すもの: 図書館・辞書・学術書の静けさ／エディトリアルデザイン（雑誌・書籍）の読みやすさ／Apple HIG の精密さ／余白の美しさ。
- **情報量は多くてよい。ただし視覚的には静かであること。** UI が情報より目立ったら失敗。
- **禁止（経緯として確定済み）**: ガラスモーフィズム／グラデーション背景／強いドロップシャドウ／青・紫を主体とした「AIサービス風」／丸いカードを大量に並べる UI／アイコン中心のデザイン／派手なアニメーション（パララックス・大きなズーム・スライド）。
- ベージュ×低コントラストの「古書風」、および生成AI的な「角丸＋薄影＋ピル型タグ＋絵文字」も引き続き不採用。

---

## 0.5 前提（技術・運用・パフォーマンス）

- **スタック**: Rails + ERB + Hotwire + importmap。**ビルドツールを入れない**。CSS は手書き（Sprockets、`tokens → base → layout → components`）。トークンは CSS カスタムプロパティ。JS は最小限。
- **権限/公開範囲**: 管理者（オーナー本人）だけが CRUD、他は閲覧のみ。**公開側にアカウント機能は作らない**。
- **パフォーマンス最優先**: web フォントは 2 本（Noto Sans JP / Shippori Mincho、`preconnect` + `display=swap`）まで。重い CDN・過剰な JS を入れない。
- `/admin/design_mocks` のデザイン案モックだけは**治外法権**（本書の規則を適用しない）。専用レイアウト・専用 CSS バンドルで隔離されている。

## 1. カラートークン（これ以外の色は使用禁止）

`app/assets/stylesheets/tokens.css` の `:root` に定義。必ず `var(--token)` で参照。

| トークン | 値 | 用途 |
|---|---|---|
| `--bg` | `#FAF8F4` | 紙（温かみのある白）。ページ全体の地 |
| `--bg-soft` | `#F3F0EA` | hover のごく淡い面。単独の背景色としては使わない |
| `--paper` | `#FFFFFF` | **操作要素だけ**を地から一段持ち上げる白（入力・ボタン・チップ・パネル） |
| `--ink` | `#202020` | 本文・見出し |
| `--ink-2` | `#575757` | 補足文 |
| `--ink-3` | `#777777` | メタ情報の下限（これ以上薄くしない） |
| `--rule` | `#E5E2DC` | 細罫（ヘアライン）。構造はほぼこれで作る |
| `--rule-mid` | `#D3CFC6` | 中間罫（入力・タグの枠） |
| `--shu` | `#B94E2F` | **朱＝唯一のアクセント** |
| `--shu-deep` | `#9B3F25` | 朱の hover |
| `--midori` | `#2F7A45` | 管理画面の「満たされている」表示だけの例外色（公開側では使わない） |
| `--on-ink` / `--on-ink-dim` / `--on-ink-rule` | — | 墨ベタ面（管理画面の一部）上の文字・罫 |

**原則**:
- **朱の使用量はページ全体の 5% 以下**。見出し・本文を朱にしない。本当に強調したい箇所（現在地・ふりがな・今日の一語・順位の上位・注釈の校正線）だけ。
- アクセントは朱の一色だけ。青・紫などの第二色は追加しない。グラデーション禁止。

## 2. タイポグラフィ（スケール対比が核）

| 役割 | 書体 | サイズ |
|---|---|---|
| **見本語**（詳細 h1） | 明朝 600 | `clamp(40px, 9vw, 92px)`。紙面いっぱい、折り返し許容 |
| ヒーロー見出し（ホーム） | 明朝 600 | `clamp(40px, 7.5vw, 82px)` |
| ページ見出し・セクション見出し | 明朝 600 | `clamp(18px, 4vw, 36px)` |
| 一覧の語 | 明朝 600 | `clamp(21px, 2.6vw, 26px)` |
| 本文・UI | **Noto Sans JP**（`--font-gothic`） | 14–16px、行間 1.8–1.95 |
| データラベル・数値・日付 | 等幅 `--font-mono` | 13–14px、字間 0.04–0.16em |

- 見出しは明朝（`--font-mincho` = Shippori Mincho）、本文はゴシック（`--font-gothic` = Noto Sans JP）。**この 2 本以外を足さない**。
- **本文・ラベルの最小サイズは 13px**（管理アノテーション・コンソール `annotate.css` は密度が高いため対象外）。
- 対比の核は「**大きな明朝の言葉 × 小さめの等幅ラベル**」。ただし旧仕様ほど極端にはせず、中間の可読サイズを許す。
- ふりがな（`rt`）は既定 `--ink-3`、**見出し語・注釈では朱**（教科書ルビの文法）。
- 太字 700 は使わない（最大 600）。

## 3. 面・罫線・枠・余白

| 要素 | 表現 |
|---|---|
| 構造の区切り | **細罫 1px `--rule`**。これが基本にして原則 |
| セクション間 | `--space-8`（80px）〜`--space-9`（120px） |
| 最大幅 | `--content-width` = **1040px**。左右の余白は広く取る（1200px は間延びしたため不採用） |
| パネル（基本情報・検索フォーム・ランキング枠・ホームの3カラム） | 1px `--rule` の枠＋`--paper` 地＋`--radius`(4px) |
| 操作要素（ボタン・入力・select・タグ・チップ） | 1px `--rule-mid` の枠＋`--paper` 地＋`--radius`(4px) |
| テーブル | ヘッダ下に 1px `--rule-mid`、行は細罫。数字は `tabular-nums` |
| フッター | **ページ全体と同じ紙色**。上端に細罫を引き、奥付のように静かに終わる |
| 背景色の面 | 基本作らない（hover で `--bg-soft` に沈む程度） |

- **角丸は `--radius`（4px）の 1 段階のみ**。それ以外の値は使わない。面やカードは角丸にしない。
- **影は原則使わない**。例外は検索フォームのカードのみ（オーナー判断。`0 2px 12px rgba(25,24,19,.05)` 程度のごく薄いもの）。
- **墨のベタ面・2px の太罫は「単語詳細」だけの文法**（§5.4）。ホーム・一覧・ランキング・統計・フッターでは使わない。
- ピル型・グラデーション禁止。カードを大量に並べる構成にしない。

## 4. インタラクション

- **transition は 200ms（`--transition`）に統一**。動きで飾らない。フェード相当のみ。
- リンク hover: 文字色が `--ink` に締まる（メタ情報のリンクは朱でもよい）。
- ボタン・ページネーション hover: **背景が `--bg-soft` にわずかに変わるだけ**。
- タグ・チップ（＝検索の絞り込みリンク）hover: **墨に反転**する。「押せる」ことを示す唯一の強い合図として残す。
- 単語詳細のシェアボタンは例外的に**墨枠＋hover 反転**（見せ場のトーンに合わせる）。
- 主要 CTA（検索実行・登録）: 墨のベタ。hover は `--shu-deep`。
- 一覧の行 hover: 背景が `--bg-soft` に変わる程度（語を朱にしない）。
- `::selection` は朱地に紙色。`:focus-visible` は朱の 2px アウトライン。

## 5. コンポーネント

### 5.0 アイコン（Phosphor Light・インライン SVG）
- **必要最小限。線画のみ。装飾目的では使わない。**
- アイコンは **Phosphor Icons の Light ウェイト**（MIT）。**必ずインライン SVG**（`app/views/shared/icons/_*.html.erb`、`icon("name")` ヘルパー）。`fill: currentColor` で墨/朱トークンに連動。**CDN・アイコンフォント・絵文字は禁止**。
- サイズは `1em` 基準（`.icon`）。テキストに添えるときは `.icon-label`。装飾アイコンは `aria-hidden`、アイコンのみの操作要素には `aria-label`。
- 出典/ライセンスは `vendor/PHOSPHOR-LICENSE`。

### 5.1 グローバルヘッダー / フッター
- ヘッダー: **高さ 64px**、下端 1px `--rule`。左=印章（朱の正方形に「言」。favicon `public/icon.svg` と共通）＋ブランド名（明朝）。右=ナビ（現在地は朱の 2px 下線）。**リンク同士の間隔を広く取り、旧仕様より情報密度を下げる**。
- **ヘッダーに常設検索は置かない**（検索はホームのヒーローと詳細検索ページが担う）。
- ナビの「検索」はプルダウン（ジャンル／索引／詳細検索）。
- フッター: **紙色のまま**、上端に細罫。4 カラムの奥付（ブランド＋説明／さがす／データを見る／サイト情報）＋等幅のコピーライト行。リンクは細いタイポグラフィで静かに並べる。

### 5.2 ホーム（主役は検索）
1. **ヒーロー**: 巨大な明朝の見出し「長い言葉を探す。」→ 短い説明 → **ページ最大の UI である検索バー**（高さ 68px・`--paper`・1px 枠・4px 角丸）→ 補助導線（詳細検索へ）
   - 見出しは末尾が「。」で右が空くため、`padding-left: 0.5em` で**視覚的な中心を右へ 0.25em 戻す**
   - リード文は `max-width: 612px`（検索バー 720px の約 85%）
2. **収録統計**: 罫線だけで仕切る 4 つの数字（箱にしない。大きな明朝の数字＋小さなラベル）。
   収録語数 / 語義数 / ジャンル数 / **平均読み字数**（サイトの性格が一目で分かる指標）
3. **今日の一語**: **全幅の帯**（左に見出し語＋読み、右に意味、見出し行の右端に「詳しく見る →」）。
   3 カラムの 1 面に入れると中身が薄く縦に間延びしたため横長に組み替えた（オーナー判断）。
   日付から決定的に選ぶ（`Date.current.jd % count`）。下の2カラムより「ほんの少しだけ」前に出すため、枠は `--rule-mid`・地は `--paper`。
   ただし**検索より目立たせない**ので、見出し語は `clamp(30px, 3.4vw, 40px)` までに留める。
4. **2 カラム**: 最近追加された単語 / 読みが長い言葉
5. 3 つの見出しにはそれぞれアイコンを添える: 今日の一語 = `sparkle` / 最近追加 = `bell_ringing` / 読みが長い = `crown`

### 5.3 一覧・検索結果（リスト UI）
- **カード UI にしない。1px の細罫だけで仕切るリスト**。共通 partial `words/_entry_row`。
- 1 件 = **言葉（明朝）／読み（等幅）／文字数・ジャンル・品詞などのタグ** のみ。
  意味と登録日は載せない（試作したが情報が多すぎて不採用）。
- hover は背景がわずかに変わる程度。

### 5.4 単語詳細（見せ場＝ここだけ活字見本帖の文法を残す）

**ここは 2026-07 の改訂対象外**。「静かな知識アーカイブ」に寄せず、旧「活字見本帖」の強い組みをそのまま維持する（オーナー判断で差し戻し済み。再度フラット化しない）。

1. パンくず → eyebrow（「見出し語」＋`No. {id}`＋別の言葉へ）→ **h1 見本語（紙面いっぱい）** → 等幅のメタ行（文字種）→ 説明 → シェア導線。**下端は墨 2px**。
2. 1 カラム。右サイドの基本情報パネル（インフォボックス）は**作らない**（試作したが不採用）。
3. **語義ごとに墨 1px 枠のパネル**（地は `--paper` の白で、紙地から一段持ち上げる）。左上に**黒ベタ白抜きの番号刻印**（`語義 01`）。中身:
   - グループ ruby（`<ruby>表層形<rt>よみ</rt></ruby>`、**rt は朱**）
   - 意味（15px・行間 2.1）
   - **該当部分つき特徴**: 注釈対象を明朝＋朱の下線（校正記号）、特徴名は**墨枠のチップ**（検索リンク・hover 反転）
   - 関連データ: ラベル列＋値列の定義グリッド、行は細罫
4. 関連語セクションの上端は墨 2px。行は一覧と同じ `words/_entry_row`。
5. 個人情報・いいね等は置かない。

### 5.5 関連データ（必ず検索への導線にする）
- **ジャンル階層**: リンク付きパンくず。**区切り `›` は朱**。末端は下線で強調。`words?genre_id=`
- **品詞・エンティティ・語種**: 細枠のタグ（hover で墨に反転）。`words?part_of_speech_id=` など
- **言語学的特徴**: 墨枠チップ。`words?linguistic_feature_id=`
- 一覧の行に添えるジャンルは `genre_lineage_label`（中分類・小分類を中黒で繋ぐ）を使う。
- 値には必ずラベルを添える。関連データを雑に並べない。

### 5.5.1 ランキング
- 1 枠 = 細罫のパネル（`--paper` 地）。**説明文（lead）は置かない**。見出し行に切替タブからの遷移先アンカーと「もっと見る」だけ。
- 順位は等幅。**1 位は朱ベタ白抜き、2〜3 位は朱枠、4 位以下は素の薄墨**で段を作る。

### 5.6 フォーム / ボタン / テーブル / ページネーション
- **ボタン既定** = `--paper` 地 + 1px `--rule-mid` + 4px 角丸 + **高さ 48px 程度**（`.btn--sm` は 34px、`.btn--icon` は 30px）。hover は背景のみ変化。塗り（墨ベタ）は主要 CTA（`.btn--primary`）のみ。朱をボタンの地に使わない。
- テキスト系入力=下線基調（focus で墨）。ただし検索フォーム内は枠付きの箱。select/textarea=細枠＋4px 角丸。
- エラーは朱テキスト＋朱枠（背景は塗らない）。
- テーブル=ヘッダ下 1px `--rule-mid`、語は明朝。ページネーション=細枠ボタン＋等幅の頁表示。

## 6. アクセシビリティ & パフォーマンス

- コントラストは WCAG AA 以上（メタ文字の下限は `--ink-3`）。`:focus-visible` を消さない。アイコンのみ操作要素に `aria-label`。タップ領域 ≥44px（padding で確保）。
- キーボードのみで一通り操作可能。ホバーでしか得られない情報を作らない。
- 重いフォント/CDN/JS を足さない。`font-display: swap`、画像 `loading="lazy"`、importmap 維持。

## 7. レスポンシブ

- モバイルファースト。検証幅 375 / 768 / 1280px。横スクロール禁止。
- 巨大見出しは `clamp()` で縮み、折り返し許容。2 カラム・3 カラム・定義グリッドは狭幅で縦積みに崩れてよい。
