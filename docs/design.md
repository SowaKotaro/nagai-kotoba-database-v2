# デザインシステム仕様「活字見本帖（Type Specimen）」（長い言葉のデータベース）

> UI を作る/変える、または雰囲気・トーンを確認するときは**必ず本書を読む**。`CLAUDE.md` からも参照。
> トークン名・数値は確定値として扱い、勝手にデフォルトの美意識へ回帰しない。

---

## 0. 思想（1行）

**「言葉そのものが主役の、活字見本帖」**。長い言葉を紙面いっぱいの明朝で組み、データは極小の等幅ラベルで添える。
構造は「墨の罫線・墨の枠」と**スケール対比**で作り、装飾では作らない。

- 参照する文法: 活字鋳造所の型見本（type specimen）、日本の校正記号・朱書き、教科書の朱ルビ。
- **禁止（経緯として確定済み）**: 生成AIのデフォルト美意識（角丸＋薄影＋ピル型タグ＋紫/青系・グラデーション・絵文字）。ベージュ×低コントラストの「古書風」も不採用（古臭くなるため）。

---

## 0.5 前提（技術・運用・パフォーマンス）

- **スタック**: Rails + ERB + Hotwire + importmap。**ビルドツールを入れない**。CSS は手書き（Sprockets、`tokens → base → layout → components`）。トークンは CSS カスタムプロパティ。JS は最小限。
- **権限/公開範囲**: 管理者（オーナー本人）だけが CRUD、他は閲覧のみ。**公開側にアカウント機能は作らない**。
- **パフォーマンス最優先**: web フォントは明朝1本（Shippori Mincho、`preconnect`+`display=swap`）だけ。重い CDN・過剰な JS を入れない。

---

## 1. カラートークン（これ以外の色は使用禁止）

`app/assets/stylesheets/tokens.css` の `:root` に定義。必ず `var(--token)` で参照。

| トークン | 値 | 用途 |
|---|---|---|
| `--bg` | `#FAF8F3` | 紙（温かみのある白） |
| `--ink` | `#191813` | 墨（本文・見出し・強罫・枠・ベタ面） |
| `--ink-2` | `#5C5950` | 補足文 |
| `--ink-3` | `#7C7969` | メタ情報の下限（これ以上薄くしない） |
| `--rule` | `#E4E1D5` | 細罫（ヘアライン） |
| `--rule-mid` | `#C9C5B4` | 中間罫（入力の下線など） |
| `--shu` | `#C43A1E` | **朱＝唯一のアクセント**。ふりがな・注釈・現在地・区切り記号・エラー・印章 |
| `--shu-deep` | `#A62E15` | 朱の hover |
| `--on-ink` / `--on-ink-dim` / `--on-ink-rule` | `#FAF8F3` / `#B6B3A6` / `#3B3931` | 墨ベタ面（フッター）上の文字・罫 |

**原則**: アクセントは朱の一色だけ。緑・青・紫などの第二色は追加しない。グラデーション禁止。

---

## 2. タイポグラフィ（スケール対比が核）

| 役割 | 書体 | サイズ |
|---|---|---|
| **見本語**（詳細 h1・今日の一語） | 明朝 600 | `clamp(38〜40px, 8.5〜9vw, 88〜92px)`。紙面いっぱい、折り返し許容 |
| 一覧の語・テーブル内の語 | 明朝 500–600 | `clamp(22px, 3.4vw, 30px)` / 15px |
| 本文・UI | OS 標準ゴシック | 13–15px、行間 1.9–2.1 |
| **データラベル**（dt・パンくず・統計ラベル・凡例・番号） | **等幅 `--font-mono`** | 10–11px、字間 0.1–0.28em |

- 「巨大な明朝の言葉 × 極小の等幅ラベル」の対比が版面を作る。中間サイズを増やさない。
- ふりがな（`rt`）は既定 `--ink-3`、**見出し語・注釈では朱**（教科書ルビの文法）。
- 太字 700 は使わない（最大 600）。

---

## 3. 面・罫線・枠（角丸・影は全面禁止）

| 要素 | 表現 |
|---|---|
| 章・ヒーローの区切り | **墨の太罫 2px**（`border-bottom: 2px solid var(--ink)`） |
| 行の区切り | 細罫 1px `--rule` |
| **主要パネル**（語義・フォームのまとまり） | **墨の 1px 枠**（`border: 1px solid var(--ink)`＝印刷物のフレーム）。語義には左上に**墨ベタの番号刻印**（`01` 等幅・白抜き） |
| テーブル | ヘッダ下に墨 2px、行は細罫。数字は `tabular-nums` |
| フッター | **墨のベタ面**（`--ink` 背景に `--on-ink` 文字）でページを締める |
| 背景色の面 | 基本作らない（hover の一段沈む程度のみ） |

**禁止**: `border-radius` 0 超、`box-shadow`、ピル型、グラデーション。

---

## 4. インタラクション

- リンク hover: 下線色または文字色が**朱**になる。
- ボタン・タグ・ページネーション hover: **墨に反転**（`background: var(--ink); color: var(--bg)`）。
- 主要 CTA（検索実行・登録）: 墨のベタ。hover は `--shu-deep`。
- `::selection` は朱地に紙色。`:focus-visible` は朱の 2px アウトライン。
- transition は color/background/border 0.15s 程度。動きで飾らない。

---

## 5. コンポーネント

### 5.1 グローバルヘッダー / フッター
- ヘッダー: 下端 **墨 1px**。左=印章（朱の正方形に「言」。favicon `public/icon.svg` と共通）＋ブランド名（明朝・字間 0.12em）。右=ナビ（現在地は**朱の 2px 下線**）＋常設キーワード検索（下線1本、枠で囲わない）。
- フッター: 墨ベタの三部構成（ブランド＋説明／メニュー／等幅小文字のコピーライトバー）。

### 5.2 ホーム（型見本の表紙）
1. ステートメント（明朝の一文）
2. **今日の一語**: 等幅の朱ラベル → 巨大な見本語（詳細へのリンク、hover 朱）→ 読み（等幅）。日付から決定的に選ぶ（`Date.current.jd % count`）
3. 検索窓（墨 2px の下線＋墨ベタの検索ボタン）
4. 収録統計: **罫線で仕切る数字の行**（箱にしない。明朝の大きな数字＋等幅ラベル）
5. 新着の単語（5.3 の見本リスト）

### 5.3 一覧（見本リスト）
- 先頭に墨 2px 罫。各行=読み（等幅 10px・字間広め）を上に、**言葉を大きな明朝**で。右端に語義数（等幅）。行は細罫区切り、hover で語が朱に。共通 partial `words/_entry_row`。

### 5.4 単語詳細（見せ場）
1. パンくず（等幅 10px）→ eyebrow（「見出し語」＋`No. {id}`）→ **h1 見本語（紙面いっぱい）** → 等幅のメタ行（文字タイプ）。下端は墨 2px。
2. **語義ごとに墨 1px 枠のパネル**。左上に番号刻印（`語義 01`・墨ベタ白抜き）。中身:
   - グループ ruby（`<ruby>表層形<rt>よみ</rt></ruby>`、**rt は朱**）
   - 意味（15px・行間 2.1・最大 46em）
   - **該当部分つき特徴**: 注釈対象を大きめ明朝＋**朱の 2px 下線**（校正記号）、特徴名は**墨枠の小ラベル**（検索リンク・hover 反転）
   - 関連データ: ラベル列（等幅 10px）＋値列の定義グリッド、行は細罫
3. 個人情報・いいね・共有は置かない。

### 5.5 関連データ（必ず検索への導線にする）
- **ジャンル階層**: リンク付きパンくず。**区切り `›` は朱**。末端は墨下線で強調、hover 朱。`search?genre_id=`
- **品詞・エンティティ**: 細枠のタグ（hover で墨に反転）。`search?part_of_speech_id=` など
- **言語学的特徴**: 墨枠チップ。`search?linguistic_feature_id=`
- 値には必ず等幅の `dt` ラベルを添える。

### 5.6 フォーム / ボタン / テーブル / ページネーション
- テキスト系入力=下線基調（focus で朱）。select/textarea=細枠。ラベルは等幅 10px。
- ボタン既定=墨 1px 枠（hover 反転）。塗り（墨ベタ）は主要 CTA のみ。朱をボタンに使わない。
- エラーは朱テキスト＋朱枠（背景は塗らない）。
- テーブル=ヘッダ等幅極小＋墨 2px、語は明朝 15px。ページネーション=墨枠ボタン＋等幅の頁表示。

---

## 6. アクセシビリティ & パフォーマンス

- コントラストは WCAG AA 以上（メタ文字の下限は `--ink-3`）。`:focus-visible` を消さない。アイコンのみ操作要素に `aria-label`。タップ領域 ≥44px（padding で確保）。
- キーボードのみで一通り操作可能。ホバーでしか得られない情報を作らない。
- 重いフォント/CDN/JS を足さない。`font-display: swap`、画像 `loading="lazy"`、importmap 維持。

## 7. レスポンシブ

- モバイルファースト。検証幅 375 / 768 / 1280px。横スクロール禁止。
- 巨大見出しは `clamp()` で縮み、折り返し許容。統計行・定義グリッドは狭幅で縦積みに崩れてよい。
