# デザインシステム仕様「紙とインク」（長い言葉のデータベース）

> UI を作る/変える、または雰囲気・トーンを確認するときは**必ず本書を読む**。`CLAUDE.md` からも参照。
> トークン名・数値は確定値として扱い、勝手にデフォルトの美意識へ回帰しない。

---

## 0. 思想（1行）

「**図鑑・辞典のような、紙とインクの手触り**」。生成りの紙に墨の文字、台紙カード、明朝の見出し。
**生成AIのデフォルト美意識（白基調＋角丸＋薄い影＋ピル型タグ＋紫/青グラデーション・絵文字）は一切採用しない。**

> 経緯: 白基調・角丸・薄影・ピルの「モダン化」を一度試作したが、**「AIデザインすぎる」ため不採用**と決定済み。同じ方向への再提案はしない。

---

## 0.5 前提（技術・運用・パフォーマンス）

- **スタック**: Rails + ERB + Hotwire(Turbo/Stimulus) + importmap。**ビルドツールを入れない**。CSS は**手書き**（Sprockets、`tokens → base → layout → components` の順で require）。トークンは **CSS カスタムプロパティ**。JS は最小限。
- **権限/公開範囲**: 管理者（オーナー本人）だけが CRUD。他のユーザーは**閲覧のみ**。**公開側にアカウント機能（サインアップ／いいね／コレクション等）は作らない**。管理者認証は従来どおり維持。
- **パフォーマンス最優先**: 重い web フォント・大きな CDN・過剰な JS を入れない。web フォントは**明朝1本だけ**（`preconnect` + `display=swap`）。デザインがローディングやユーザビリティを悪化させないことを常に優先する。
- **適用範囲**: 既存画面（ホーム／公開一覧・詳細／検索／管理 CRUD／ログイン）のリスタイル。新機能・スキーマ変更はしない（→ §8）。

## 1. カラートークン（これ以外の色は使用禁止）

`app/assets/stylesheets/tokens.css` の `:root` に定義。必ず `var(--token)` で参照し、直書きしない。

| トークン | 値 | 用途 |
|---|---|---|
| `--paper` | `#F7F2E7` | ページ全体の背景 |
| `--paper-card` | `#FFFDF8` | 台紙カードの面（本体より少し明るい） |
| `--ink` | `#2B2B28` | 本文・見出し |
| `--ink-muted` | `#5A5440` | 補足文 |
| `--ink-faint` | `#8A8168` | メタ情報（最も弱い文字） |
| `--hairline` | `#D8CFBB` | すべての境界線（1px） |
| `--border-strong` | `#B7AC8E` | 強調境界（二重罫線の上段・入力の下線） |
| `--vermillion` | `#B33A2E` | **注釈（該当部分）・エラー・ブランド印章専用**の朱 |
| `--moss` | `#5F6B3E` | 言語学的特徴チップの背景専用 |
| `--moss-strong` | `#4E5832` | チップの hover |
| `--moss-fg` | `#F3F5EA` | moss 上の文字 |

**禁止**: 紫・青・藍系、グラデーション全般。`--vermillion` をボタン・リンクの汎用色に流用しない。

## 2. タイポグラフィ

| 用途 | フォント | 備考 |
|---|---|---|
| 見出し（h1〜h4）・オーバーライン・セクションラベル・ブランド | 明朝 `--font-mincho`（Shippori Mincho） | ラベルは 11px＋字間広め（0.25〜0.3em）で「版面のラベル」らしく |
| 本文・UI | `--font-gothic`（OS 標準ゴシック） | web フォント追加禁止 |
| 番号・韻・文字タイプなどの記号列 | `--font-mono` | 10〜12px |

- **主役は「長い言葉」**。見出し語は `clamp()` で可変（例 `clamp(24px, 5vw, 34px)`）、折り返しを許容する。
- 太字 700 は使わない（最大 600）。
- web フォントは明朝のみ `preconnect` + `display=swap`。CSP は現状未適用のため変更不要（適用時は `fonts.googleapis.com` / `fonts.gstatic.com` を許可）。

---

## 3. 面・罫線・角丸・影

| 項目 | 値 |
|---|---|
| 角丸 | **`border-radius: 0` 固定（全面禁止）**。ピル型タグも禁止 |
| 影 | **使わない（`box-shadow` 全面禁止）** |
| 境界線 | 1px、`--hairline`（強調のみ `--border-strong`） |
| **主要コンテンツの輪郭** | **台紙カード**＝`--paper-card` 背景 + 1px `--hairline` 罫線。背景と本文の境界を明確にする |
| リストの区切り | 一覧・検索結果などの**行**は 1px 罫線で区切る |
| ラベル表 | メタ情報は **1px 隙間のグリッド**（`gap:1px` の下地に `--hairline` を見せて罫線にする）で標本ラベル風に |
| ページヘッダ | 見出し語ブロックは**二重罫線**（`--border-strong` 1px ＋ 少し離して `--hairline` 1px）で辞書の版面らしく |
| 内側パディング | 台紙カード 16–20px |

**禁止**: 影・角丸で「浮いた箱」を作らない（輪郭は必ず 1px 罫線＋紙色の面で出す）。一覧の行を個別カードの羅列にしない。余白は `--space-*` のスケール変数で管理する。

## 4. アイコン（インライン SVG）

- **アイコンフォント・絵文字は使わない**。Tabler(outline) 系の stroke SVG を必要な分だけ**インライン**（`currentColor` / `viewBox 0 0 24 24` / `stroke-width` 1.75 前後）。partial（`app/views/shared/icons/`）にまとめる。
- 使う最小セット: `search` / `chevron-down` / `plus` / `x` / `arrow-left` / `arrow-right`。むやみに増やさない。
- サイズ 14–18px、色は `--ink-faint` か `--vermillion` のみ。

## 5. コンポーネント仕様

### 5.1 グローバルヘッダー
- 下端 1px `--hairline`。背景は `--paper` のまま（面を分けない）。
- 左: **ブランド印章**（`--vermillion` の正方形に `--paper` の「言」。インライン SVG。favicon `public/icon.svg` と共通の意匠）＋ブランド名「長い言葉のデータベース」（明朝 17px・字間 0.1em）。
- 右: ナビ（`一覧` / `検索`、ログイン時のみ `管理` / `ログアウト`）。**現在地のナビは 2px の `--border-strong` 下線**で示す。
- 右端に**常設のキーワード検索**（`q` → `/search`）。**枠で囲わず下線1本**（ピル型検索窓は禁止）。

### 5.1.2 サイトの「製品らしさ」を支える定番要素（必ず置く）
- **パンくず**（公開ページ: `ホーム › 単語一覧 › {語}` など。`shared/_breadcrumbs`）。
- **フッター三部構成**: 概要（ブランド名＋説明文。「収録データはすべて運営者が精査のうえ登録」と品質を明示）／メニュー（一覧・検索・管理者ログイン）／コピーライトバー。
- **ホーム＝サービスの顔**: ヒーロー（ブランド名＋説明＋大きめの下線検索窓）→ 収録統計（語数・語義数・ジャンル数。1px 罫線グリッド＋明朝の数字）→ 新着の単語（罫線区切りの行＋「すべて見る」）。空のトップページにしない。

### 5.2 単語詳細（見せ場）
1. **ページヘッダ**: オーバーライン（明朝 11px・字間 0.3em。「見出し語」＋右端に `No. {id}` の mono）→ `h1` 見出し語（明朝・`clamp()`）→ `char_type_pattern` のメタ行。下端は**二重罫線**（§3）。
2. **語義ごとに台紙カード**（§3。同音異義語で複数並ぶ）。右上に `語義 n` の mono 通し番号。各カード:
   - 読み: `surface` にその語義の `reading` を**グループ ruby**（`<ruby>表層形<rt>よみ</rt></ruby>`）。読みは必ずテキスト（画像化しない）。
   - 意味: `WordSense.meaning`。
   - **該当部分つき特徴**: セクションラベル（明朝 11px・字間広め）＋ 特徴名の**苔チップ**（`--moss`、角丸なし、**同じ特徴での検索へのリンク**）＋ `target`（`target_reading`）を**朱の点線下線**（`border-bottom: 1px dotted var(--vermillion)`）で注釈。
   - **関連データのラベル表**（1px 罫線グリッド。§5.3 のコンポーネントをセルに収める）。
3. 個人情報・投稿者名・いいね・共有は**置かない**（§0.5）。

### 5.3 関連データの扱い（重要）
関連データは「テキストを並べるだけ」を禁止し、**必ず専用コンポーネント＋検索リンク**にする:
- **ジャンル階層**: リンク付きパンくず `大 › 中 › 小`。区切りは `›`（`--border-strong`）。**各階層が `search?genre_id=` へのリンク**（上位を選べば配下ごと絞り込める）。末端（小分類）は `--ink`＋weight 500 で少し強調。
- **品詞・エンティティタイプ**: **四角いタグ**（1px `--hairline` 罫線＋`--paper-card`。角丸にしない）。`search?part_of_speech_id=` / `search?entity_type_id=` へのリンク。
- **言語学的特徴**: 苔チップ。`search?linguistic_feature_id=` へのリンク。
- 読みの文字数（`n文字`）・韻（mono）はテキストで良いが、ラベル（`dt`）を添えて何の値かを常に明示する。

### 5.4 リスト行 / テーブル（一覧・検索結果・管理一覧）
- 罫線区切りの行、または台紙カード。ホバーは薄い背景変化程度に留める。

### 5.5 ボタン
- 既定はアウトライン（1px `--border-strong` + `--ink` 文字、角丸なし）。
- 塗りボタン（`--ink` 背景＋`--paper` 文字）は**主要 CTA（登録・検索実行）だけ**。`--vermillion` をボタンに使わない。

### 5.6 フォーム
- 入力は**下線基調**（1px `--hairline`、フォーカスで `--border-strong`）またはラベル表のセル内入力。ピル型の枠は禁止。
- バリデーションエラーは**朱テキストのみ**（背景を塗らない）。
- ジャンルの大→中→小 依存ドロップダウンは既存 Stimulus（`genre-cascade`）、行の動的追加/削除は `nested-form` を流用。

### 5.7 ページネーション
- 「前へ / n ページ / 次へ」を罫線・テキストで。矢印はインライン SVG。

## 6. アクセシビリティ & パフォーマンス

- 画像に `alt`、アイコンのみの操作要素に `aria-label`。`outline` を消さず `:focus-visible` で再定義。コントラストは WCAG AA（4.5:1）以上を ink 系トークンで担保。
- キーボードのみで一通り操作できること。ホバーでしか得られない情報を作らない。タップ領域 ≥44×44px（見た目は変えず padding で確保）。
- **重いフォント/CDN/JS を足さない**。`font-display: swap`、画像 `loading="lazy"`、importmap 維持。
- 機械チェック可能な禁止事項: `border-radius` が 0 超 / `box-shadow` / グラデーション / 絵文字 / 紫・青・藍系の色。

---

## 7. レスポンシブ

- **モバイルファースト**。検証幅は最低 **375 / 768 / 1280px**。横スクロール前提のレイアウトは禁止。
- 見出し語は `clamp()` 可変＋折り返し許容。ラベル表（1px グリッド）は狭幅でも 2 カラムを基本とし、崩れる場合は 1 カラムへ。
- タップ領域は §6 のとおり。

## 8. スコープと将来（今回やらないこと）

- **今回**: 既存画面のリスタイルのみ。新機能・スキーマ変更はしない。
- **将来 Issue 候補**: 形態素分解テーブル＋ **MeCab + mecab-ipadic-neologd** による読み自動取得・**ワードクラウド** ／ SEO 強化（可読 URL・JSON-LD・sitemap）／ OGP 画像自動生成 ／ PWA ／ サジェスト検索。
- **今後も作らない**: 公開側のアカウント前提機能（いいね・コレクション・ランキング・個人化）。
- **ダークモード**: 「紙とインク」が核のため単純な色反転は不可。やるなら「夜の書斎」の配色を別途設計してから（優先度低）。
