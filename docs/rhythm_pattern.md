# rhythm_pattern 変換仕様

`word_senses.reading`（読み・かな）をヘボン式ローマ字へ写像した文字列。
韻（リズム）での検索・分類の軸に使う。生成は Ruby 側（[`app/models/rhythm_pattern.rb`](../app/models/rhythm_pattern.rb)）で行い、
`WordSense` の `before_validation` で自動セットする（手入力させない）。SQL では生成できないため生成カラムにはしていない。

## 方針の決定
- **ローマ字方式: ヘボン式（Hepburn）**。発音に忠実で一般的・可読性が高く、韻（響き）の比較に向くため。
  - `し→shi` / `ち→chi` / `つ→tsu` / `ふ→fu` / `じ→ji` / `しゃ→sha`
- **長音: 母音をそのまま展開**。書かれた読みに忠実で実装がシンプル。
  - `とうきょう→toukyou`、`おう→ou`
  - 長音符 `ー`（`ｰ` も NFKC で吸収）は**直前の母音を繰り返す**（`カレー→karee`、`コーヒー→koohii`）。

## 変換手順
1. **正規化**: `unicode_normalize(:nfkc)` で半角カナ・合成濁点を畳み、`tr("ァ-ヶ", "ぁ-ゖ")` でカタカナをひらがなへ寄せる。
   これで変換表をひらがな1本に統一し、カタカナ語も同じ規則で処理できる。長音符 `ー` はこの範囲外なので保持される。
2. 先頭から走査し、**拗音・外来音（2文字）を直音（1文字）より先に**照合してローマ字へ写像する。
3. **促音 `っ`**: 次の音の先頭子音を重ねる（`がっこう→gakkou`）。ただし `ch` の前は `t` を置く（`まっちゃ→matcha`、`こっち→kotchi`）。
   次が母音始まり／末尾の場合は重ねない。
4. **長音符 `ー`**: それまでの出力の末尾側にある母音を1つ繰り返す（直前に母音が無ければ無視）。
5. 変換表に無い文字（記号など）は**そのまま通す**。

## 具体例
| reading | rhythm_pattern | 備考 |
| --- | --- | --- |
| `さくら` | `sakura` | 直音 |
| `しんぶん` | `shinbun` | 撥音 `ん` は `n` |
| `がっこう` | `gakkou` | 促音は子音重ね＋長音は母音展開 |
| `まっちゃ` | `matcha` | 促音＋`ch` は `t`（`tcha`） |
| `とうきょう` | `toukyou` | 拗音＋長音の母音展開 |
| `カレー` | `karee` | `ー` は直前母音を繰り返す |
| `ﾗｰﾒﾝ` | `raamen` | 半角カナも NFKC で吸収 |
| `つづき` | `tsuzuki` | `づ→zu` |

## 割り切り（境界・仕様上の判断）
- **撥音 `ん` は常に `n`**。`b`/`p`/`m` の前でも `m` にしない（`しんぶん→shinbun`、`さんぽ→sanpo`）。
  検索・パターン照合の軸として**規則性を優先**したため。母音・`y` の前でもアポストロフィは付けない
  （`しんや` と `しにゃ` はともに `shinya` に畳まれる。可逆な翻字ではなくリズム軸としての割り切り）。
- **`ぢ→ji` / `づ→zu`**（ヘボン式）。`を→o`、`ゐ→i`、`ゑ→e`。
- **`ゔ`（＝`ヴ`）→ `vu`**、`ゔぁ→va` など外来音にも対応。
- 変換表に無い文字はそのまま通すため、記号・数字が読みに混ざるとローマ字にそのまま残る。
- `nil`・空文字は空文字列を返す。

判定順（2文字→1文字）や促音・長音の処理順を変えると結果が変わりうるため、`RhythmPattern` の実装順序を維持すること。
