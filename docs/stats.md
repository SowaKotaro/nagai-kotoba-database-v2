## 1. デザインコンセプト「蔵版目録」

活字見本帖のメタファーを統計ページまで延長する。実際の活字鋳造所の見本帳が巻末に
字種・在庫の集計表（蔵版目録）を載せていたように、統計ページはその位置づけ——
**本編は言葉が主役、この頁だけは等幅数字と罫線が主役**になる帳簿の一枚。

### チャートの文法（design.md の統計への翻訳ルール）

- **識別は色でなくラベルと位置で持つ**（横棒 + 行ラベル）。多色カテゴリカルは作らない。
  朱一色方針と色覚アクセシビリティ（色相に依存しない）を同時に満たす。
- **朱は「最」にだけ使う**: 最頻値・当月・最多セル。それ以外は墨。
- **量の濃淡は朱チント**（`color-mix(in srgb, var(--shu) N%, transparent)`。索引の `.kana-cell--heat` と同じ式）。
- ベースラインは墨 2px（テーブルのヘッダ罫と同じ文法）。グリッドは `--rule` のヘアライン。
- 数字は等幅 `tabular-nums`。値の直書きは端点・最大値のみ（全点に数字を振らない）。
- **すべての棒・セル・扇がリンク**（`search?reading_length=` 等）。design.md §5.5
  「関連データは必ず検索への導線」を統計にも適用する。
- チャートライブラリは導入しない。インライン SVG + 最小限の JS（描画ヘルパ程度）で描く。
- 角丸・影・グラデーション禁止は統計でも同じ。面の塗りは単色（朱チントの平塗り）のみ。

## 2. 紙面構成（確定）

上から順に。§番号は朱の等幅ラベル（`§1 — 看板` のように内容語を添え、装飾番号にしない）。

### 0. ページヘッダ + 数字の壁（蔵版目録）

- eyebrow「DATA — 蔵版目録」+ 明朝の大見出し「収録統計」+ リード文。
- 「集計 ◯時点 ／ 毎日再集計」の等幅キャプション（奥付の文法。キャッシュ実装と一致させる）。
- **数字の壁**: 4群 × 各5指標の定義リスト（罫線区切り・箱にしない）。出せる数値は全部出す方針。

| 群 | 指標 | 出どころ |
|---|---|---|
| 規模 | 収録語・語義・別表記・同音異義の組・のべ読み文字 | count ／ `HAVING count>1` ／ `SUM(reading_length)` |
| 読みの長さ | 平均・中央値・最頻・最長・平均モーラ | `reading_length` / `mora_count` 集計 |
| 文字と音 | 頭文字の種類（44/46 のようなカバレッジ表記）・末尾文字の種類・カタカナのみ%・漢字を含む%・「ー」を含む% | `first_char` / `last_char` DISTINCT、`char_type_pattern`、`reading` LIKE |
| 分類と歩み | 今月の新収録・ジャンル数・特徴数・1日あたり平均・開帳からの日数 | count / `created_at` |

- 濁音を含む%・拗音を含む%・平均語義数/語なども同様に出せる（増やしてよい）。
- カバレッジ表記（44/46字）は「しりとりで詰む字がある」という読み物に繋がるので採用。

### §1 看板: 級数見本（ワードクラウドの見本帖化）

- 収録語を MeCab で形態素に分け、よく現れる部品（「選手権」「シンドローム」等）を
  **活字の大きさ＝頻度**で組む。渦巻きランダム配置の一般的ワードクラウドは使わず、
  活字見本帳の級数表のように**文字組（ベースライン揃えの折返し）**で並べる。
- 明朝 600。最頻のみ朱。肩に等幅の出現数。語を押すとその形態素を含む語の検索へ。
- **実装上の制約**: 本番/CI に MeCab が無い（メモリ `bulk-word-reading-import` 参照）。
  形態素頻度は**管理側/ローカルで事前集計してテーブルまたはキャッシュに置く**設計が必要。
  Phase 1 に含めず、事前集計の仕組みとセットで別 PR。

### §2 音のはじまりとおわり（50音表ヒートマップ + 行×行ヒートマップ）

- **50音表の先頭文字・末尾文字ヒートマップを併載**（2026-07-19 オーナー指示）。
  索引ページの kana-grid / 朱ヒートをそのまま再利用し、`first_char` / `last_char` の
  正規化済みカウント（ひらがな読みはカタカナへ畳む）で塗る。セル押下で絞り込みへ。
- **五十音円環（Issue 62）は不採用**。一目で何を示すか読めないため、
  頭文字の行 × 末尾文字の行の **11×11 マトリクス**（ア行〜ン行 × ア行〜ン行の正方形）に置き換え。
- セルは朱チントの濃淡（`--fill` カスタムプロパティ + `color-mix`）。最多セルのみ数字を白抜き直書き、
  他は hover で表示。セル押下でその組み合わせの語一覧へ。
- **行見出し・列見出しはどちらも1文字**（「ア行」ではなく「ア」。2026-07-23 変更）。
  行見出しはマスに寄せて**右揃え**。「行」の意味は見出し・隅見出し・ホバーの文言で補う。
- 右側に読み物キャプション（「カ行→ラ行が最多」「ンの列は行き止まり」「ワ行は過疎地」）。
- 将来拡張: セル押下で 46×46 の字単位ヒートマップに展開する設計も可。
- データは `GROUP BY first_char, last_char` の1クエリ + Ruby 側で行に集約。

### §3 読みの長さ（波形塗りバー・単位切替）

- 索引ページで温存していた**波形塗りバー**をここで起用（採用確定）。
  棒の上辺を正弦波にする（SVG path をサーバ側で生成。振幅 ~2.6px・1周期）。
- 塗りは朱チント 38%、最頻値のみ朱ベタ。直書きラベルは最頻値と両端のみ。
- **文字数／モーラ数の切替**を墨枠セグメントボタンで付ける
  （`reading_length` と `mora_count`。モーラ=拗音の小書きを除いた拍、シャ=1拍・ー=1拍）。
- 棒押下で `search?reading_length=n`（モーラ側は `mora_count`）へ。
- **30以上は「30+」のまとめ棒**（2026-07-20 変更。まれな超長語が散らばると横軸が間延びするため。
  `SiteStatistics::DISTRIBUTION_OVERFLOW_MIN`）。文字数側のまとめ棒は `reading_length_min=30` の
  範囲検索へリンク、モーラ側は範囲検索パラメータが無いためリンクなし（hover 反転もしない）。

### §4 収録の推移（株価チャート式）

- **累計の折れ線（墨 2px + 朱チント10%の面）+ 下段に「出来高」= 週別新収録の棒（墨、直近のみ朱）**。
  株価チャートの文法で「今月の勢い」を語る。累計を棒に重ねる二軸グラフは採らない（一軸原則）。
- **期間切替**（全期間／6か月／3か月）を墨枠セグメントで。端点に朱の点 + 累計値の直書き。
- 見出し脇に「1,238語 +220(今月)」の現在値表示。
- データは `GROUP BY` した `created_at` の週次/月次集計。

### §5 ジャンル別の語義数（Plotly サンバースト + 連動棒グラフ）

- **Plotly サンバーストで実装**（2026-07-19 オーナー指示。B案「紙面割付」→自前SVGサンバースト
  を経て確定）。内側から大分類・中分類・小分類。`branchvalues: total`・`sort: false`・`rotation: 90`。
- **扇クリックでその部分が全体となって展開**（Plotly 標準のドリルダウン）。
  末端（小分類）の扇クリックは `words?genre_id=` の絞り込みへ遷移。
- **`maxdepth: 2` で常に「中心+2階層」だけ描画**（2026-07-19 変更）。初期表示は大分類+中分類で、
  小分類は中分類を押したときだけ現れる。小分類が多くてもラベルが読め、戻るときの再描画も軽い。
- 右は**1クリック=1階層の段階展開**（2026-07-19 変更。全中分類×小分類の積み上げ横棒の
  一括展開は、小分類が多いと縦に伸びすぎ・クリック不能になるため廃止）:
  大分類の積み上げ縦棒 → 押すとその大分類の**中分類の積み上げ縦棒** → さらに押すと
  その中分類の**小分類を墨枠タグ+件数の一覧**（`.tag` + `.genre-hub__count` を流用、
  `words?genre_id=` への検索導線）で表示する（`app/javascript/controllers/genre_sunburst_controller.js`）。
- パレットは朱の濃淡のみ（`sunburstcolorway` + `extendsunburstcolorway`）。
- Plotly はこのページ専用: コントローラが接続時に `vendor/javascript/plotly.min.js` を1度だけ挿入する。

### §6 ことばの出どころ（語種 + エンティティ型）

- **語種構成は「活字ケース」= ワッフルチャート**（10×10 マス、1マス≒1%）。
  文選箱に活字が詰まっている見立て。凡例は 朱ベタ（外来語）／墨ベタ（漢語）／朱チント（和語）／
  白+墨枠（混種語）の4値——明度差があるので色覚にも安全。帯グラフより「詰まっている」感が出る。
- **エンティティ型は squarified ツリーマップ**（2026-07-19 オーナー指示でタグクラウド→
  flex タイルを経て確定）。面積が語義数に比例（`StatsHelper#stats_entity_treemap`、座標は%）、
  塗りは朱チントの濃淡。セル押下で絞り込み検索へ。
  - 罫は各セル `--rule-mid` の 1px（2026-07-23 変更。墨 1px だと隣り合う辺が二重になって重い）。
  - **全体の 3% 未満の型は末尾から「その他」へ畳む**（`TREEMAP_MIN_SHARE`）。読めず押せない
    マスを作らないため。畳んだ「その他」自体が 3% に届くまで次に小さい型も巻き込み、
    畳む相手が 1 つだけなら面積が変わらないのでそのまま残す。「その他」は絞り込み先が無いので
    リンクにせず `--bg-soft` の面（`stats/_entity_cell`）。

### §7 音の内訳（母音スペクトル + 頭子音）

- **母音スペクトル**: 横=語頭からの拍位置、縦=母音（ア〜オ段）構成比の積み上げ面グラフ。
  `vowel_pattern` / `rhythm_pattern` から集計。「語尾に近づくほどウ段・オ段が増える=
  『〜ション』『〜シンドローム』の音」が読める、このDBならではの図。
  帯は墨/朱の明度5段 + 右端に段名を直書き（識別を色に依存させない）。
  - 攻めた図なので、分かりにくければ「母音構成比の帯1本」に落とす退路を持つ。
- **頭の子音ランキング**: 読み第1拍の子音（ヘボン式、`rhythm_pattern` 先頭）を横棒で。

### §8 ことばの見どころ（言語学的特徴・見本つきランキング）

- 特徴を件数バーで並べつつ、各行に**実例語の該当部分に朱 2px 下線**を添える
  （単語詳細の校正記号の文法をランキングに持ち込む。`word_sense_features.target` が使える）。
- 特徴名は墨枠チップ（hover 反転）→ `search?linguistic_feature_id=`。

### 奥付（フッター直前）

- 「本表の数値は毎日再集計されます ／ データは CC BY 4.0 で利用できます」の等幅一行。

## 4. 追加アイデア（将来。モック未掲載）

1. **しりとり完走解析（学術寄り・看板級）**: 頭文字→末尾文字は有向グラフなので、
   オイラー路の存在判定で「収録語だけでしりとりは最長何語つながるか」を厳密計算できる。

   等幅で並べた頻度表。パターン自体がタイポグラフィなので見本帖と相性が良い。
4. **統計 OGP カード**: 統計ページ共有時に主要数字入りの OGP 画像を自動生成（被リンク獲得の実利）。
5. **研究者向け生データ導線**: 集計値の CSV/JSON ダウンロードを奥付に（CC BY 4.0 と整合）。
   「学術利用できる日本語長語データセット」の独自ポジション。
6. **Zipf プロット**: 形態素頻度の log-log 図。載せるなら奥付近くに小さく（一般読者には渋い）。

## 5. 掲載統計カタログ（構想の全量。紙面採用分を含む）

### 5.1 全語義で成立する統計（データが常にある）

| 統計 | 元データ | 見せ方（採用状況） |
|---|---|---|
| 読みの長さ分布 | `reading_length` | **§3 波形塗りバーで採用** |
| モーラ数分布 | `mora_count` | **§3 の切替で採用**。文字数とズレる語（拗音・長音が多い語）は**ランキングページ `/rankings` で採用** |
| 最長語ランキング | `reading_length` | ホーム＋**ランキングページ `/rankings`** に実装済み。統計ページでは数字の壁の「最長」から導線 |
| 頭文字・末尾文字の分布 | `first_char` / `last_char` | **§2 行×行ヒートマップで採用**（円環は不採用） |
| 文字種構成 | `char_type_pattern` | 数字の壁（カタカナのみ%・漢字を含む%）で採用。型見本ランキングは将来 |
| 表記の長さ・一字あたりの読み | `words.surface` × `reading_length` | **`/rankings` で採用**（「表記が長い」「一字が重い」） |
| 拗音・促音・長音符・濁点 | `reading` / `mora_count` | **`/rankings` で採用**（読みの文字数と拍数の差・`ー` の数・濁音の数） |
| 母音・韻 | `vowel_pattern` / `rhythm_pattern` | **§7 母音スペクトル + 頭子音で採用** |
| 収録数の推移 | `created_at` | **§4 株価チャート式で採用** |

### 5.2 アノテーション依存の統計（付与率が上がったら）

- **ジャンル分布**: **§5 紙面割付ツリーマップで採用**。大→中→小のドリルダウン。
- **品詞構成比**: 名詞への極端な偏り自体が見せられる事実。少数派は全件列挙でも成立。
- **エンティティタイプ内訳**: **§6 タグクラウドで採用**。珍しい型は件数 + 実例を添える。
- **語種構成**: **§6 活字ケースで採用**。多対多を活かした混種語率・語種の組み合わせ頻度が独自統計。
- **言語学的特徴ランキング**: **§8 見本つきランキングで採用**。「特徴てんこ盛り語」ランキングは **`/rankings` で採用**。

### 5.3 派生カラムとのクロス集計（このDBならではの層・将来）

- ジャンル × 平均読み長（どの分野の言葉が長いか）
- 語種 × 読み長（外来語は長いのか、漢語複合語が最長か）
- 語種 × 文字種パターン（カタカナ語=外来語の対応がどれくらい崩れるか）
- 特徴 × 語種（連濁・音便は和語に集中するはず、という定説の検証）
- エンティティ × ジャンル（作品名→芸術 等の対応マトリクス）

### 5.4 特徴データの中身（target / target_reading）の分析（将来）
