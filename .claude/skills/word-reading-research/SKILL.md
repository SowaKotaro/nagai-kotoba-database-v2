# 単語の読み調査（オフライン）

nagai-kotoba-database-v2 の一括登録を助けるための、**アプリから切り離した**調査タスク。
管理者が登録したい単語（表層形）の集合を渡すので、各語について **最も一般的な表記とその読み（カタカナ）** を
調べ、下記スキーマの **JSON だけ** を出力する。この JSON を step2 の取り込み欄に貼ると、MeCab の暫定読みと
突き合わせて最終的な読みを確定できる。

## 大原則
- **MeCab の読みは受け取らない／参照しない。** 単語（表層形）だけから独立に読みを判定する。
  目的は「MeCab が誤る読み（例: 「花は桜木人は武士」の 人＝ヒト を MeCab は ジン と誤読）」を
  独立ソースとして正すこと。MeCab の答えを先に見ると追認バイアスで誤りを見逃すため、絶対に混ぜない。
- **確信が持てない語は Web 検索で裏取りする**（固有名詞・新語・ことわざ/四字熟語の難読・人名地名など）。
  最低2つの独立した情報源で一致を確認し、一致しなければ confidence を下げ alternatives に併記する。
- 迷ったら**でっち上げない**。confidence を `low` にし、根拠を notes に書く。

## 入力
- 箇条書き（`1.` `-` `・` などの行頭記号は無視）／改行区切り／JSON 配列のいずれか。1行＝1語（表層形）。
- 表層形はスペースを含みうる（例「Dead by Daylight」）。行全体を1語として扱う。

## 調査手順（1語ずつ）
1. まず知識から、その語の**最も一般的な表記**（`surface`）と**読み**（`reading`）を決める。
2. 次のいずれかに当たる語は **WebSearch で裏取り**する:
   - 固有名詞・作品名・人名・地名・ブランド・新語/ネットスラング
   - ことわざ・慣用句・四字熟語で読みが割れやすいもの（訓読み/音読み、連濁、熟字訓）
   - 複数の一般的な読みがある語（例: 日本＝ニホン/ニッポン、上手＝ジョウズ/ウワテ/カミテ）
3. 読みが複数あり得るときは、最も一般的なものを `reading` に、他を `alternatives` に入れる。
4. `confidence` を付ける: `high`=定番/裏取り済み、`medium`=妥当だが未検証、`low`=不確実。

## 読みの表記ルール（MeCab と揃えて突き合わせやすくする）
- **カタカナ**で書く。長音は「ー」（例: プロテスタンティズム、カレー）。
- **助詞は表記どおりのかな**にする（MeCab -Oyomi の慣例）: は→ハ、へ→ヘ、を→ヲ。
  （例「花は桜木人は武士」→ ハナハサクラギヒトハブシ）
- 促音「ッ」、拗音「ャュョ」はそのまま小書き。記号・空白・中黒は読みに含めない。
- 数字は文脈上の一般的な読みをカタカナで（例「四十七士」→ シジュウシチシ）。

## 出力（これだけを出す）
- 説明文を付けず、**下記スキーマの JSON を1つだけ** コードブロックで出力する（step2 にそのまま貼れるように）。
- `input` は渡された原文のまま（step2 が表層形で突き合わせるキー）。`surface` は最も一般的な表記
  （多くは input と同じだが、異体字・旧字などを一般表記へ正規化した場合は変えてよい）。
- スキーマの定義と例は同ディレクトリの [`schema.json`](schema.json) / [`example.json`](example.json) を参照。

```json
{
  "version": "1",
  "words": [
    {
      "input": "花は桜木人は武士",
      "surface": "花は桜木人は武士",
      "reading": "ハナハサクラギヒトハブシ",
      "alternatives": [],
      "confidence": "high",
      "notes": "ことわざ。人=ヒト(MeCabはジンと誤読しやすい)"
    }
  ]
}
```
