# 単語のアノテーション調査（オフライン）

nagai-kotoba-database-v2 のアノテーション作業を助けるための、**アプリから切り離した**調査タスク。
管理画面の「調査用データの書き出し」が出力する JSON（対象語 + マスタ一覧）を受け取り、各語について
下記スキーマの **提案 JSON だけ** を出力する。この JSON を管理画面の「提案 JSON の取り込み」に貼ると
下書き（annotation_proposals）として保存され、オーナーがコンソールで1語ずつ確認・修正して承認する。

## 大原則
- **提案はあくまで下書き。** 公開データには直接入らないので、書けることは全部書く。ただし
  **不確実な内容をでっち上げない**。迷ったら `confidence: "low"` にして `notes` に迷いの内容を書く。
- **マスタは渡された一覧から選ぶ。** `masters` にある名前と**一字一句同じ表記**で書く（取り込み側は
  名前でマスタを解決する）。適切な選択肢が無いときだけ新しい名前を書き、ジャンルなら `genre_new: true`
  を立てる（新設候補としてオーナーに提示される）。
- **固有名詞・作品名・新語は WebSearch で裏取りする。** 意味の記述・ジャンル判定・エンティティ判定の
  根拠を最低1つの信頼できる情報源で確認する。裏取りできなければ confidence を下げる。

## 入力
管理画面が書き出す JSON:
- `words`: 調査対象。`word_id`（取り込みのキー。**そのまま返す**）・`surface`（表層形）・`reading`（読み）
- `masters`: 選択肢。`genres`（大→中→小のパスの配列）・`entity_types`・`parts_of_speech`・
  `word_origins`・`linguistic_features`（いずれも名前の配列）

## 調査手順（1語ずつ）
1. 表層形と読みから語を特定する。曖昧な語（同名の作品・人物が複数あるなど）は WebSearch で確定し、
   どれを採ったか `notes` に書く。
2. **意味（`meaning`）**: 公開ページのリード文になり得るトーンで、1〜2文・自己完結で書く。
   文体は「〜。」の常体（例:「漫画『ONE PIECE』の登場人物。麦わらの一味の船長。」）。
3. **ジャンル（`genre_path`）**: `masters.genres` のパスから最も適切な**末端（小分類）まで**を選ぶ。
   適切な小分類が無ければ、既存の大・中分類の下に新しい小分類名を置き `genre_new: true` を立てる。
4. **エンティティ（`entity_type`）**: 固有名詞なら該当する種類を `masters.entity_types` から。
   一般語なら省略する。
5. **品詞（`part_of_speech`）**: `masters.parts_of_speech` から。ほとんどの収録語は名詞・固有名詞系。
6. **語種（`word_origins`）**: 和語・漢語・外来語・混種語などを `masters.word_origins` から（複数可）。
   読みと表記から機械的に判定できることが多い（カタカナ外来語、漢字熟語=漢語 など）。
7. **別表記（`variants`）**: 一般に流通する別表記があれば `surface`（+読みが変わるなら `reading`）で。
   無ければ省略。
8. `confidence` を付ける: `high`=裏取り済みか定番 / `medium`=妥当だが未検証 / `low`=不確実。
   `low` のときは必ず `notes` に「何に迷ったか・何を確認してほしいか」を書く。

## 出力（これだけを出す）
- 説明文を付けず、**下記スキーマの JSON を1つだけ** コードブロックで出力する（取り込み画面にそのまま貼れるように）。
- `word_id` は入力の値をそのまま返す（取り込みの突き合わせキー）。
- 値の無いキーは省略してよい。スキーマの定義と例は同ディレクトリの
  [`schema.json`](schema.json) / [`example.json`](example.json) を参照。

```json
{
  "version": "1",
  "proposals": [
    {
      "word_id": 123,
      "surface": "モンキー・D・ルフィ",
      "meaning": "漫画『ONE PIECE』の主人公。麦わらの一味の船長。",
      "genre_path": ["文化", "漫画・アニメ", "ONE PIECE"],
      "genre_new": false,
      "entity_type": "キャラクター名",
      "part_of_speech": "固有名詞",
      "word_origins": ["混種語"],
      "variants": [{ "surface": "ルフィ", "reading": "ルフィ" }],
      "confidence": "high",
      "notes": ""
    }
  ]
}
```
