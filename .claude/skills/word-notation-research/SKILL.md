# 最も一般的な表記の下調べ（登録前・オフライン）

nagai-kotoba-database-v2 に登録しようとしている単語の候補について、**登録する前に**
「最も一般的な表記」だけを確定するための、アプリから切り離した調査タスク。
リポジトリ直下の入力ファイルを読み、調査結果を出力ファイルに書き出す（**ファイル入力 → 調査 → ファイル出力**）。

## フロー
1. **入力**: リポジトリ直下 `inputs_word_tmp.txt` を読む。1行＝1語（表層形）。
   行頭の番号/記号（`1.` `-` `・`）は無視。空行は無視。表層形はスペースを含みうる（例「Dead by Daylight」）＝**行全体を1語**として扱う。
2. **調査**: 各語の最も一般的な表記を決める（下記の基準）。確信が持てない語は WebSearch で裏取りする。
3. **出力**: リポジトリ直下 `outputs_word_tmp.txt` に書き出す（**上書き**）。

## 調べること / 調べないこと
- **調べる**: 最も一般的な **表記（surface）だけ**。異体字・旧字・冠称/敬称・説明句・送り仮名や漢字/かなの揺れを正規化する。
- **調べない**: 読み（→ `word-reading-research`）、意味・ジャンル・品詞・語種・別表記（→ `word-annotation-research`）。

## 大原則
- **判断基準は [`docs/annotation-guidelines.md`](../../../docs/annotation-guidelines.md) §3（表記の選び方）が正。** オーナー個人の感覚ではなくこの基準で判定する。
- 次のいずれかに当たる語は **WebSearch で裏取り**する（辞書見出し・公式表記・用例数で判断。最低2つの独立ソースで確認）:
  - 固有名詞・作品名・人名・地名・ブランド・新語/ネットスラング
  - ことわざ・慣用句・四字熟語（表記が割れやすいもの）
  - 異体字・旧字・送り仮名の揺れがある語
- **迷ったらでっち上げない。** 確信が持てなければ入力の表記を保ち、`confidence` を下げて注記に理由を書く。

## 表記の選び方（annotation-guidelines §3 準拠・要点）
- 冠称・敬称・説明句を除いた**本体**を採る。
- 異体字・旧字は常用の字体へ寄せる。ただし固有名詞で**公式表記が旧字/異体字**ならそれを優先する。
- 正式名称と通称は「最も一般に流通する方」を採る。
- 句読点・中黒・前後の空白は原則入れない（慣用でその形が定着している場合のみ残す）。

## 出力フォーマット（`outputs_word_tmp.txt`）
先頭に、各語の**最も一般的な表記**を入力と同じ番号で1行ずつ並べる。
**この上部リストはそのままコピペして登録／読み調査の入力に使うので、表記そのものだけを書く**
（`（元: …）` などの注記や補足を行末に付けない）。

続けて、入力から表記を変えた語があれば `--- 入力から表記を変更した語 ---` の見出しの下に
`<番号>. <元の表記> → <変更後の表記>（変更理由）` を1行ずつ列挙する（変更が無ければこの節は省く）。

最後に `--- 注記 ---` の下へ、1語ずつ確信度と裏取りの根拠（変更した理由・迷った点）を書く。
`confidence` は high / medium / low。

例:

```
# 最も一般的な表記（inputs_word_tmp.txt の調査結果）
1. 天上天下唯我独尊
2. 花は桜木人は武士
3. プロテスタンティズムの倫理と資本主義の精神
4. ゴールドマン・サックス

--- 入力から表記を変更した語 ---
4. ゴールドマンサックス → ゴールドマン・サックス（中黒を追加）

--- 注記 ---
1. 仏教語。この表記が定着。confidence: high
2. ことわざ。読点なしのこの形が一般的。confidence: high
3. M.ウェーバーの著作の邦題（岩波文庫・大塚久雄訳）。この表記が標準。confidence: high
4. 金融機関。Wikipedia・コトバンク見出しは中黒入り。confidence: high
```

## この後の流れ
確定した表記リスト（`outputs_word_tmp.txt` 上部）は、そのまま一括登録の入力や、
`word-reading-research` の入力 `inputs_reading_tmp.txt` に渡せる（表記 → 読み → 登録 → アノテーションの前段）。
