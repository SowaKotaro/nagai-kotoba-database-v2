# 最も一般的な表記の下調べ（登録前・オフライン）

nagai-kotoba-database-v2 に登録しようとしている単語の候補について、**登録する前に**
「最も一般的な表記」を確定し、あわせて**その語が立項に足るか**を判定するための、
アプリから切り離した調査タスク。
`research/` 配下の入力ファイルを読み、調査結果を出力ファイルに書き出す（**ファイル入力 → 調査 → ファイル出力**）。

## フロー
1. **入力**: `research/inputs/notation.txt` を読む。1行＝1語（表層形）。
   行頭の番号/記号（`1.` `-` `・`）は無視。空行は無視。表層形はスペースを含みうる（例「Dead by Daylight」）＝**行全体を1語**として扱う。
2. **調査**: 各語の最も一般的な表記を決める（下記の基準）。確信が持てない語は WebSearch で裏取りする。
   裏取りの過程で得た情報から、**立項に足る語か**も判定する（下記）。
3. **出力**: `research/outputs/notation.txt` に書き出す（**上書き**）。

## 調べること / 調べないこと
- **調べる**:
  - 最も一般的な **表記（surface）**。異体字・旧字・冠称/敬称・説明句・送り仮名や漢字/かなの揺れを正規化する。
  - **立項に足る語かどうか**（収録の4原則への適合度＝立項スコア）。
- **調べない**: 読み（→ `word-reading-research`）、意味・ジャンル・品詞・語種・別表記（→ `word-annotation-research`）。
  立項判定は「4原則を満たすか」だけを見るのであって、**語義やジャンルの記述までは踏み込まない**。

## 大原則
- **判断基準は [`docs/annotation-guidelines.md`](../../../docs/annotation-guidelines.md) §3（表記の選び方）が正。** オーナー個人の感覚ではなくこの基準で判定する。
- 次のいずれかに当たる語は **WebSearch で裏取り**する（辞書見出し・公式表記・用例数で判断。最低2つの独立ソースで確認）:
  - 固有名詞・作品名・人名・地名・ブランド・新語/ネットスラング
  - ことわざ・慣用句・四字熟語（表記が割れやすいもの）
  - 異体字・旧字・送り仮名の揺れがある語
- **迷ったらでっち上げない。** 確信が持てなければ入力の表記を保ち、`confidence` を下げて注記に理由を書く。

## 表記の選び方（annotation-guidelines §3 準拠・要点）
- 冠称・敬称・説明句を除いた**本体**を採る。
- 異体字・旧字は常用の字体へ寄せる。ただし固有名詞で**公式表記が旧字/異体字**ならそれを優先する。
- 正式名称と通称は「最も一般に流通する方」を採る。
- 句読点・中黒・前後の空白は原則入れない（慣用でその形が定着している場合のみ残す）。

## 立項に足るかの判定（annotation-guidelines §1・§2 準拠）

表記を裏取りすると、その語が**そもそも立項に値するか**の材料は自然に集まる。それを捨てずに評価する。
**収録の4原則**（§1: 公然性・妥当性・慣用性・同定可能性）に照らし、**立項スコア**（§2）を 1〜5 で付ける。

| スコア | 定義 | 扱い |
|---|---|---|
| 5 | 4原則すべてを明確に満たす。辞書・百科事典・公式媒体で即確認できる | 上部リストに残す |
| 4 | すべて満たすが、裏取りが単一ソースなど確認が薄い | 上部リストに残す（根拠を注記に） |
| 3 | 1つの原則にグレーがある（慣用性が新しすぎる、同定に文脈が要る 等） | 上部リストに残し、**要オーナー判断**と注記に明記 |
| 2 | 1つの原則を明確に欠く、または2つ以上がグレー | **上部リストから外す** |
| 1 | 複数の原則を明確に欠く（内輪語・即興造語・検証不能） | **上部リストから外す** |

- スコアは「語の有名さ」ではなく **基準への適合度**。マイナーでも4原則を満たせば 5 になり得る（§2）。
- 判定の手掛かりは §5（知名度・一般性の判定目安）に従う。辞書・百科事典の見出しがあれば強い根拠、
  **独立した複数ソース**で確認できれば十分な根拠、単一の投稿・単一のファンサイトしか無ければグレー（3以下）。
- **情報が不足していて裏が取れない語**は、想像で埋めずスコアを下げる。「確認できなかった」ことは
  スコアと confidence の**両方**に反映させる（下記）。
- よくある減点例:
  - 冠称・肩書き・説明句が付いたまま（妥当性）→ まず表記の正規化で救えるなら救う。救えないなら減点。
  - 語ではなく文・修飾句の断片（妥当性）。
  - 検索しても第三者の公開媒体に存在を確認できない（公然性）。
  - 一過性のバズ・単一投稿でしか使用例が無い（慣用性）。
  - 何を指すか一意に定まらない曖昧な名称（同定可能性）。

### confidence との関係
`confidence` は**この調査結果全体の確からしさ**を表す。表記の確信度だけでなく、
**立項判定の確信度も判断材料に含める**。

- 表記が辞書見出しで確定でき、立項スコアも自信を持って付けられる → `high`
- 表記は確定したが、立項スコアに迷いがある（グレー＝3 など） → `medium`
- **情報が不足して裏が取れない**、または**立項に値しない疑いが強い**（スコア2以下） → `low`。
  注記に「何を確認してほしいか」を書く（§5）。

## 出力フォーマット（`research/outputs/notation.txt`）
先頭に、各語の**最も一般的な表記**を入力と同じ番号で1行ずつ並べる。
**この上部リストはそのままコピペして登録／読み調査の入力に使うので、表記そのものだけを書く**
（`（元: …）` などの注記や補足を行末に付けない）。
**立項スコア2以下の語はここに入れない**（下記の疑義節へ落とす）。番号は入力のものを維持し、
欠番はそのままにする（オーナーが元の行と突き合わせられるようにするため）。

続けて次の3節を置く。

- `--- 入力から表記を変更した語 ---`: `<番号>. <元の表記> → <変更後の表記>（変更理由）`。変更が無ければ省く。
- `--- 立項に疑義がある語 ---`: **立項スコア2以下**の語を `<番号>. <語>（entry_score N。どの原則を・なぜ欠くか）`
  の形で列挙する。**捨てずに残す**のは、オーナーが判断を覆して復活させられるようにするため。無ければ省く。
- `--- 注記 ---`: 1語ずつ `entry_score` と `confidence`、裏取りの根拠（変更した理由・迷った点・
  確認できなかったこと）を書く。`confidence` は high / medium / low。
  スコア3以下には**どの原則がグレーか**を必ず添える（§2）。

例:

```
# 最も一般的な表記（research/inputs/notation.txt の調査結果）
1. 天上天下唯我独尊
2. 花は桜木人は武士
3. プロテスタンティズムの倫理と資本主義の精神
4. ゴールドマン・サックス
6. 関西電気保安協会

--- 入力から表記を変更した語 ---
4. ゴールドマンサックス → ゴールドマン・サックス（中黒を追加）
6. 関西電気保安協会さん → 関西電気保安協会（敬称を除去）

--- 立項に疑義がある語 ---
5. とてつもなく大きな（entry_score 1。妥当性を欠く: 語ではなく修飾句の断片で、見出し語の単位を成さない）

--- 注記 ---
1. 仏教語。この表記が定着。entry_score: 5（辞書見出しあり）/ confidence: high
2. ことわざ。読点なしのこの形が一般的。entry_score: 5 / confidence: high
3. M.ウェーバーの著作の邦題（岩波文庫・大塚久雄訳）。この表記が標準。entry_score: 5 / confidence: high
4. 金融機関。Wikipedia・コトバンク見出しは中黒入り。entry_score: 5 / confidence: high
5. 修飾句の断片。表記以前に語として成立しない。entry_score: 1 / confidence: low（立項不可の疑い。要オーナー判断）
6. CM で知られる企業名。公式サイトで表記を確認。entry_score: 4 / confidence: high
```

## この後の流れ
確定した表記リスト（`research/outputs/notation.txt` 上部）は、そのまま一括登録の入力や、
`word-reading-research` の入力 `research/inputs/reading.txt` に渡せる（表記 → 読み → 登録 → アノテーションの前段）。
立項スコア2以下の語は上部リストから外してあるので、**後工程にはふるいを通った語だけが流れる**。
疑義節の語を復活させるかどうかはオーナーが判断する。

なお、登録後の `word-annotation-research` も `entry_score` を付ける（§2）。こちらは**登録前**の
足切りで、判定材料は表記の裏取りで得た範囲に限られる。語義・ジャンルまで踏まえた最終的な立項判定は
アノテーション段階で行われるので、ここでのスコアは**暫定値**として扱ってよい。
